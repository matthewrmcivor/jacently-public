# MVP Goals - Jacently

## Project Overview

Jacently is a mobile-first global event discovery platform that helps users find local events happening nearby. The MVP focuses on creating a simple, intuitive map-based experience for discovering events within a user's proximity.

## Core MVP Objectives

### 1. Location-Based Discovery

- **Goal**: Enable users to find events within 5-150 miles of their location
- **Success Metric**: Users can successfully view events on a map within their selected radius
- **Priority**: Critical (P0)

### 2. Map-Centric Interface

- **Goal**: Provide an interactive map view as the primary discovery method
- **Success Metric**: Map loads quickly (< 3 seconds) and displays event pins accurately
- **Priority**: Critical (P0)

### 3. Event Filtering

- **Goal**: Allow users to filter events by time and attributes
- **Filters**: "Now", "Today", "This Week", "Free", "Family-Friendly"
- **Success Metric**: Filters work correctly and update map/list view in real-time
- **Priority**: High (P1)

### 4. Event Details

- **Goal**: Display comprehensive event information
- **Content**: Photos, dates, times, location, contact info, description
- **Success Metric**: Users can access all essential event details from detail page
- **Priority**: High (P1)

### 5. Event Creation (Sponsor Portal)

- **Goal**: Enable event sponsors to create and manage their events
- **Success Metric**: Sponsors can successfully post events that appear on the map
- **Priority**: High (P1)

### 6. Mobile-First Experience

- **Goal**: Optimize for mobile devices (phones and tablets)
- **Success Metric**: App functions smoothly on mobile browsers, responsive design works across screen sizes
- **Priority**: Critical (P0)

## MVP Scope Boundaries

### In Scope (MVP)

- ✅ Map view with event pins
- ✅ List view alternative
- ✅ Basic event filters (time-based and attribute-based)
- ✅ Event detail pages
- ✅ Event creation form for sponsors
- ✅ Geolocation-based search
- ✅ Mobile-responsive design

### Out of Scope (Post-MVP)

- ❌ User accounts and authentication (Phase 2)

## User Personas

### Primary Persona: Casual Event Seeker

- **Name**: "Local Explorer"
- **Age**: 25-45
- **Tech Comfort**: Moderate to High
- **Goals**:
  - Find interesting events happening nearby
  - Discover new activities and venues
  - Plan weekend activities
- **Pain Points**:
  - Fragmented event information across multiple sources
  - Difficulty finding events that match interests
  - Lack of real-time event availability

### Secondary Persona: Event Sponsor

- **Name**: "Local Business Owner"
- **Age**: 30-60
- **Tech Comfort**: Low to Moderate
- **Goals**:
  - Promote events to local audience
  - Increase event attendance
  - Manage event listings easily
- **Pain Points**:
  - Limited reach for event promotion
  - Complex event posting processes
  - Difficulty reaching target demographics

## Success Criteria

### Technical Success

- [ ] Map loads and displays events within 3 seconds
- [ ] App works on iOS Safari, Chrome Android, and desktop browsers
- [ ] Geolocation accuracy within 100 meters
- [ ] No critical bugs in core user flows

### User Experience Success

- [ ] Users can find at least 5 events within 10 miles in under 2 minutes
- [ ] Event detail pages contain all essential information
- [ ] Filters work intuitively without user confusion
- [ ] Mobile experience feels native and responsive

### Business Success

- [ ] Prototype demonstrates clear value proposition
- [ ] Stakeholders can visualize full product vision
- [ ] Technical feasibility validated for Phase 2
- [ ] Clear path forward for MVP development
